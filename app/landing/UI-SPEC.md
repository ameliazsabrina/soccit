# Soccit Landing — AAA Game Landing Redesign UI Design Contract

> Visual and interaction contract for a AAA game-styling redesign of `/landing`. Generated by gsd-ui-researcher. Targets the gap between "motion-polished web layout" and "looks like a AAA game landing page". Supersedes the prior polish-pass contract for the same file.

---

## 1. Scope & Goal

### 1.1 Goal

Transform the existing motion-polished landing page into something that reads, on first impression, as a **AAA game landing page**: an alive, atmospheric, world-built experience with a persistent game-style HUD, gamification cues, kinetic typography, sound design, and one continuous match-day cinematic. NOT incremental polish — a soul transplant.

### 1.2 What the page is today

A fully working, lint-clean, motion-rich linup at `app/landing/`:

- `page.tsx` — server shell with `PageLoader` + `ScrollProgress`
- `landing-hero.tsx` — GSAP SplitText headline ("CALL THE" + "GAME." gradient), 3D score reveal with gold glow, cursor-reactive player card tilt, magnetic CTA linking to `/matches`, scroll cue
- `how-it-works.tsx` — `300vh` pinned 3-panel sequence with step indicator (Call Score → Pitch Subs → Lock It In)
- `cards-gallery.tsx` — horizontal-scroll TCG gallery with velocity-driven rotationY/skewX, center-card scale staging, position-themed glows
- `leaderboard-teaser.tsx` — digit-roll counters, dominant podium, shimmer prize pool
- `cta-section.tsx` — "READY" outline + "TO PLAY?" solid split headline, aurora mesh, magnetic CTA with click ripple, rotating logo ring, footer
- `page-loader.tsx` — sessionStorage-gated branded loader
- `scroll-progress.tsx` — framer-motion scroll progress bar
- `animated-counter.tsx`, `digit-roll-counter.tsx` — reusable counters
- Hybrid motion: GSAP for scroll/pin/scrub/SplitText, framer-motion for reveals/hovers/counters

Build state on authoring date: `npm run build` and `npm run lint` clean (0 errors, 6 warnings — none in `/landing`).

### 1.3 What this redesign is

- A soul transplant: replace "web page with motion" with "game screen"
- A persistent game-style HUD frame (`#hud-frame`) overlaid across all sections
- A continuous stadium environment built from CSS/SVG (no video, no WebGL, no heavy 3D engines)
- Gamification cues (XP bar, level chip, rank, pack teases, unlocks)
- Kinetic typography: scoreboard flip, glitch, outline-into-fill, scrollstory
- Sound design tied to scroll beats and the existing `/sounds/` system
- One continuous "match day" scroll choreography, not 5 disconnected sections

### 1.4 What this redesign is NOT

- Replacing the 5 narrative beats — keep them
- Breaking the flat-radius (`0px`) rule
- Adding video files, WebGL, Three.js, 3D engines, or any runtime heavy dependency
- Adding new fonts outside Space Grotesk + Inter + Material Symbols
- Using `lucide-react` (the installed `1.23.0` is stale — Material Symbols only)
- Splitting the cyan token into "blue vs gold" — `--cyan` is gold `#dba111`; honor that everywhere

### Redesign typography treatments
- **Scoreboard flip.** Hero + Panel A scores animate as stadium scoreboard flips — each digit rolls `rotateX: 90deg → 0deg` over 0.6s on a per-digit stagger (60ms). Wrap each digit in `.scoreboard-digit` (see section 14).
- **Outline-into-fill (scrollytelling).** Section heading words begin as `.text-outline` and fill in (via `color: transparent → currentColor` with a `mix-blend-mode: overlay` color wash) at their ScrollTrigger entry. This reads as the title "rendering in" like a game-key UI. Apply on every section eyebrow + heading pair.
- **Glitch line (single-use).** The hero's `GAME.` word gets a one-frame `::before` clone offset `clipPath: inset(0 0 50% 0)` + `transform: translateX(-2px)` opacity 0.3 in rose and gold stacks on entrance — reads like a stadium-screen signal cut. Resist using it more than once across the page.
- **Display-size range widened** to `text-9xl` (112px) for the hero on `lg:` screens to match the "HERE IS THE GAME" pole-vaulting read of AAA title-type.

> **The page is built *on top of* a background. AAA game landings are built *inside* an environment.**

Every other gap (HUD, particles, kinetic type, sound, gamification, card treatment) is a downstream consequence. ## 9. Bold Redesign Direction — One Continuous Stadium

### 9.1 The pitch (1-2 sentences)

**The entire page is one match day, scrolled.** A persistent game-style HUD frame overlays all five sections, the page lives *inside* one continuous stadium environment (CSS/SVG only — fog, floodlights, crowd silhouettes, drifting embers, scanlines), and scroll progresses through five acts of one match-day cinematic — arrival → call score → pitch the XI → lock in → hall of fame → "kick off" — each dissolving into the next, not starting and ending.

### 9.2 Three creative pillars

**Pillar A — Place before content.** Every section sits inside the stadium environment. The environment is one continuous world (sky → stands → pitch → tunnel → hall of fame → tunnel exit) that morphs as you scroll, not 5 separate backgrounds. Atmosphere is always-on: fog drifts, embers float, floodlights bleed, scanlines tick.

**Pillar B — Persistent game HUD.** A `#hud-frame` overlays the entire page edge-to-edge: corner brackets (TL gold, TR gold, BL purple, BR purple), a scanline film-grain overlay, a LIVE chip top-right that pulses, a "PLAYERS NOW" counter top-center, a match-timer bottom-center ("MATCH DAY · 02:14 · ACT 3/5"), an XP bar bottom-left that fills as you scroll, and a pack-rarity streak chip bottom-right. These elements never leave the screen — they ARE the game framing.

**Pillar C — One scroll, five acts.** Scroll = match day. Act 1 (ARRIVAL) hero dissolves into Act 2 (CALL THE SCORE) panel A via a fog wipe. Panel A → Panel B via a floodlight sweep. Panel B → Card vault (PACK OPEN) via a card-pack-tear transition. Card vault → Podium via a confetti burst carry-over. Podium → Finale via a stadium-tunnel exit. The whole page is ONE scroll choreography with persistent atmosphere and HUD, not 5 separate sections.

### 9.3 The atmosphere layer (always-on, behind all sections)

A `#stadium-atmosphere` `fixed inset-0 z-0` element composed of CSS/SVG layers (all CSS-only, no video):

1. **Sky gradient.** `linear-gradient(to bottom, #0a0a0a 0%, #0a1628 40%, #122b1e 70%, #081a12 100%)` — dark void → night sky → pitch turf → pitch deep. Locked to `--background`/`--pitch-*` tokens. Subtle parallax with scroll (`backgroundPosition` shifts `0%` → `20%`).
2. **Stadium fog.** Three radial-gradient strips at `bottom: 20%`, `bottom: 40%`, `bottom: 60%` — `rgba(3,70,148,0.04)` → transparent. Slow `translateY` drift (12s, `ease-in-out`, alternate) via `@keyframes fog-drift`. Locked to purple token, opacity ≤ 5%.
3. **Floodlight cones.** Two SVG `<polygon>` triangles anchored `top: 0 left: 15%` and `top: 0 right: 15%`, fill `url(#floodlight-gradient)` (linear white 8% → transparent), `clip-path: polygon(45% 0, 55% 0, 100% 100%, 0 100%)`. Slow `opacity` pulse (4s, 0.6 → 1.0 → 0.6). Gives the page top two visible light sources like a stadium under floodlights.
4. **Drifting embers.** ~24 `4px × 4px` gold/rose particles (`background: var(--cyan)` or `var(--rose)` at 40% opacity) positioned with `Math.random()` once on mount, animated upward with `@keyframes ember-rise` (12s linear, `translateY(110vh) → -10vh` + slight `translateX` drift + `opacity 0.5 → 0`). Stagger `animationDelay` 0–12s. Respect `prefers-reduced-motion` (static at 30% opacity, no motion).
5. **Scanline film-grain.** A `repeating-linear-gradient(0deg, transparent 0px, transparent 2px, rgba(248,250,252,0.015) 2px, rgba(248,250,252,0.015) 3px)` overlay at 4% opacity. Tight 3px scanline reads as a CRT/HUD-screen overlay. Inert — pure CSS background, no JS.
6. **Vignette.** `radial-gradient(circle at 50% 50%, transparent 50%, rgba(0,0,0,0.35) 100%)` top layer — pulls the eye to center, frames content like a camera lens. Optional density bump during Panel C "lock" beat.
7. **Light leak transitions.** Between sections: a full-bleed `radial-gradient` wash (purple 12% → transparent) that scrubs opacity 0 → 1 → 0 across a 10vh scroll window via ScrollTrigger. Each section hand-off gets a 1-frame color wash instead of a hard opacity crossfade.

### 9.4 The persistent HUD frame (`#hud-frame`)

## 11. Buildable With Existing Stack — Analysis

Every effect in this spec is achievable with the locked stack. No video, no WebGL, no Three.js, no heavy 3D engines. Here is the mapping:

| Effect | Implementation | Stack |
|--------|----------------|-------|
| Stadium atmosphere (fog, floodlights, embers, scanlines, vignette) | `fixed inset-0` divs with CSS gradients, SVG `<polygon>` cones, `repeating-linear-gradient`, `@keyframes ember-rise`. No JS for atmosphere except ember `Math.random()` on mount. | CSS + SVG + minimal JS |
| Persistent HUD frame | One `#hud-frame` component in `page.tsx`. Corner brackets via SVG. Chips via styled divs. Match-timer + XP bar via `ScrollTrigger` global progress callbacks. | React + GSAP ScrollTrigger |

## 12. Constraints to Respect (Hard Rules)

1. **Flat radius `0px` everywhere.** Including HUD chips, particles, avatars, confetti rectangles, stamp border. NO `border-radius` on any new element.
2. **Brand colors only.** `--purple #034694`, `--cyan #dba111` (gold), `--rose #ed1c24`, `--gold #dba111`, `--bronze #cd7f32`, `--pitch-*` tokens. Rarity greens/blues (section 5.2) are scoped to the pack-vault rarity-glow utilities ONLY — they do not enter the global 60/30/10 split.
3. **The cyan token is GOLD `#dba111`.** Do not split it into "blue vs gold". Everywhere the spec says "gold" or "cyan", it means `#dba111`.
4. **Fonts:** Space Grotesk (display + tech) + Inter (body) + Material Symbols (icons). No new fonts.
5. **No `lucide-react`.** The installed `1.23.0` is stale. Material Symbols only for all icons (existing `<link>` in `app/layout.tsx`).
6. **No video files, no WebGL, no Three.js, no heavy 3D engines.** All effects per section 11.
7. **No new runtime dependencies.** Use existing GSAP 3.15, framer-motion 12.42.2, Next.js Image, React 19.
8. **`prefers-reduced-motion` honored everywhere.** Atmosphere static, boot intro 0.3s, no shake, no confetti, no sound on beats.
9. **Sound respects existing mute.** Check the same `localStorage` flag the global sound delegation uses.
10. **Dark mode is priority.** All effects tuned for `#0a0a0a` backdrop. Light mode gets 0.8× opacity fallback.
11. **Keep the 5 narrative beats.** Hero → HowItWorks → CardsGallery → LeaderboardTeaser → CTASection. No new sections, no removed sections.
12. **No rounded corners.** (Repeated because it's the most-violated rule in polish passes.) Confetti rectangles are `4px × 8px` sharp. HUD chips are sharp-cornered. Stamp is sharp-cornered. Particles are sharp squares.
